# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Frontend monitoring library (npm: `@rilog-development/rilog-lib`). It intercepts HTTP requests (fetch + XHR + axios), click events, input changes, console errors/warnings, and custom log messages — then batches them in localStorage and ships them to a backend (local file server, self-hosted endpoint, or Rilog cloud).

## Commands

```bash
yarn build          # generate-secrets.js → tsc → dist/
yarn test           # jest (all tests)
yarn test --testPathPattern=<file>  # single test file
yarn format         # prettier over src/**/*.ts
yarn lint           # tslint -p tsconfig.json
yarn patch          # npm version patch → format + changelogen + git add
yarn release        # npm publish --access public
```

`yarn build` runs `scripts/generate-secrets.js` first, which reads `.env` and writes `src/secrets.ts` (gitignored). Build will succeed with defaults if `.env` is missing.

### Data flow

1. `rilog.init(config?)` — if `config.deployServer` is set: POST `/connection/init` → receives `access_token` + `recording` flag. Otherwise resolves immediately with `recording: true`.
2. fetch/XHR/axios interceptors fire `onRequest` → entry added to `QueueArray`; `onResponse` → dequeue + build `IRilogRequestItem`.
3. Each event passes through `pushEvents` → stored in `localStorage['rilevents']`.
4. Flush triggers when: array > 10 events **and** queue is empty, localStorage nears capacity (4.5 MB), events exceed 2 MB, or long-poll timer fires (12 s inactivity).
5. Events sorted by timestamp, AES-encrypted with salt, sent via `saveEvents`.
6. On `result === 'success'` the localStorage key is cleared.

### Send priority

`localServer` → `selfServer` → `deployServer` (Rilog cloud). Checked in this order inside `sendEvents` and `onBeforeUnload`.

### Request lifecycle & timeout

- Requests enter `QueueArray` on `onRequest`.
- A per-request `setTimeout(300 000 ms)` auto-evicts stale entries and records a timeout event.
- `dequeueNotResolved(olderThanMs)` is used during flush to capture in-flight requests older than `LONG_TIMER_LIMIT`.

### Last Gasp (page unload)

`onBeforeUnload` uses `navigator.sendBeacon` for `localServer`/`selfServer`, and `fetch` with `keepalive: true` for `deployServer`.

## Public API

```ts
import rilog from '@rilog-development/rilog-lib';

rilog.init(config?);          // config IS TRilogInitConfig directly — no wrapper object
rilog.wrapAxios(axiosInstance);
rilog.logData(data, { label });
rilog.interceptRequestAxios(config);   // manual axios alternative to wrapAxios
rilog.interceptResponseAxios(response);
```

### TRilogInitConfig key fields

| Field | Type | Purpose |
|---|---|---|
| `localServer` | `ILocalServerConfig` | Send to rilog-local-server (`appName` + `url` required; `url` is the base URL — local or remote) |
| `selfServer` | `ISelfServer` | Send to custom backend (`url` = full POST endpoint URL, any path) |
| `deployServer` | `IDeployServerConfig` | Send to Rilog cloud (`key` required) |
| `ignoredRequests` | `string[]` | URLs to skip entirely |
| `sensetiveRequsts` | `string[]` | URLs where headers+body → `"sensetive"` |
| `sensetiveDataRequests` | `string[]` | URLs where only body → `"sensetive"` |
| `headers` | `string[]` | Allowlist of headers to capture (default: none) |
| `localStorage` | `string[]` | Allowlist of localStorage keys to snapshot per request |
| `disableFetchInterceptor` | `boolean` | Skip `window.fetch` patching |
| `disableXHRInterceptor` | `boolean` | Skip `XMLHttpRequest` patching (use when `wrapAxios` covers all requests) |
| `disableClickInterceptor` | `boolean` | Skip click listener |
| `disableConsoleInterceptor` | `boolean` | Skip console wrap |
| `disableInputInterceptor` | `boolean` | Skip input focusout capture |
| `onPushEvent` | `(event) => void` | Callback per intercepted event |
| `onSaveEvents` | `(events) => void` | Callback on flush |
| `meta` | `TExternalInfoMeta` | `{ environment, branch, framework, platform }` |

## Code style

- **TypeScript strict mode**, target `es5`, `commonjs`.
- **Prettier**: `printWidth: 200`, `tabWidth: 4`, single quotes, trailing commas.
- **tslint** (`tslint.json`) enforced before publish (`prepublishOnly`).
- Classes + interfaces for all core logic. Adapters implement interfaces (`IAxiosAdapter`, `IFetchAdapter`).
- `@logMethods` decorator on public methods of `Rilog` for dev-only console debug output.
- **Intentional typos** in identifiers (`sensetive`, `libruaryRequest`, `queque`) — do not rename; they appear in types, constants, and method names throughout the codebase.
- Interfaces prefixed `I`, types prefixed `T`, enums prefixed `E`.
- No external runtime dependencies except `crypto-js`.

## Key constraints

- Library targets browsers only (`window.fetch`, `localStorage`, `document`).
- `initFetchInterception` patches `window.fetch` globally — only call once per page.
- `EVENTS_ARRAY_LIMIT` (10) flush only triggers when `requestsQueue.isEmpty()` — never flush mid-flight.
- Max localStorage budget: 4.5 MB total, 2 MB for events buffer.
- `ERilogEvent` enum must stay in sync with the backend event type numbers.
- `DEFAULT_LOCAL_URL` (`http://localhost:3030`) is the fallback when `localServer.url` is not provided — but `url` is required in `ILocalServerConfig`, so this constant is used as the fallback in `sendEvents`/`onBeforeUnload`.

## Build & release

- `changelogen` auto-generates `CHANGELOG.md` on `yarn patch` / `yarn minor`.
- Secrets (`BASE_URL`, `LOCAL_BASE_URL`, salts) injected by `scripts/generate-secrets.js` from `.env` → `src/secrets.ts`.
- `DEFAULT_LOCAL_URL` is a non-secret constant exported from `constants.ts` directly.

## Commit message rule

**After every task, always output a suggested git commit message** in [Conventional Commits](https://www.conventionalcommits.org/) format:

```
<type>(<optional scope>): <short imperative description>
```

Common types: `feat`, `fix`, `refactor`, `chore`, `docs`, `test`, `perf`, `build`.
Keep the subject line under 72 characters. No period at the end.

Example:
```
feat(interceptor): add per-request timeout with auto-eviction
```
